Catalog Database Excel Worksheets

Error Finder Sheet - Download

The goal of this sheet is to automate the identification of part numbers with missing values that do not meet the required data fields to become sellable.

  • Catalog Database
    • Columns A through I represent a self-generated catalog database.
    • Includes multiple vendors for two product classes — Food Product and Office Supply.
    • Some products contain missing values, which must be audited to ensure data integrity.
  • Total Errors
    • This count sums errors from Unidentified Class and Missing UPC sections.
    • Provides a quick view of how many audits are required to maintain data integrity standards.
  • Unidentified Class
    • Columns M through S list products with classifications not found in the Catalog Database.
  • Missing UPC
    • Columns U through AA return products with missing UPC values within the Catalog Database.
  • Notes
    • Additional functionality can be built on these examples, such as:
    • Identifying vendors with the most errors — helps detect larger data issues or temporarily disable problematic vendors.
    • Finding products with mismatched values — useful for tracking rolling changes and vendor stock consistency.
    • Suggesting substitutions — provides the next best match for products with errors.

Search and Sourcing Sheet - Download

The goal of this sheet is to search for a product by its name and return its availability from multiple vendors.

  • Catalog Database
    • Columns A through I represent a self-generated catalog database that includes some of the same products from different vendors.
    • Only two product classes are included to ensure ease of demonstration.
    • Note: This example database is built under the presumption that it meets 100% of our data integrity requirements.
  • Search Function
    • Column L contains the search functionality and example search terms.
    • L3 serves as the search text box where product names can be entered.
    • L7 through L14 display the best-matching results, including vendors with varying inventory levels (from high to none).
  • Search Results
    • Columns N through W return products based on the Search Function.
    • Results are sorted by QTY (inventory) in descending order for quick visibility of available stock.
    • Returned products are color-coded to indicate inventory confidence and availability:
      • Green – High Confidence
      • Yellow – Medium Confidence
      • Grey – Zero Inventory
    • With sales history data, this feature can be expanded to include recommended add-on products related to the searched item.
